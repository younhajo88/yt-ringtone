# 유튜브 벨소리 웹앱 설계

## 목표

이 Windows PC에서 개인용 웹 도구를 실행하고, 휴대폰이나 PC 브라우저에서 접속해 유튜브 검색 결과 또는 유튜브 URL로부터 30초짜리 MP3 벨소리 파일을 만들 수 있게 한다.

1차 버전은 편집된 MP3 파일을 만드는 데 집중한다. Android, iPhone, PC에서 해당 파일을 벨소리로 지정하는 과정은 앱 밖에서 사용자가 직접 처리한다.

## 확정된 범위

- Windows PC에서 웹 서버를 실행한다.
- PC 또는 휴대폰 브라우저에서 `http://PC_IP:PORT` 주소로 접속한다.
- 로그인, 비밀번호, 계정, 비밀 링크 인증은 추가하지 않는다.
- 키워드로 유튜브를 검색한다.
- 직접 입력한 유튜브 URL도 받는다.
- 검색 결과를 짧은 목록으로 보여준다.
- 선택한 오디오를 PC 서버에서 다운로드한다.
- 오디오를 MP3로 변환한다.
- 브라우저에서 오디오를 미리 들을 수 있게 한다.
- 사용자가 시작 시간을 선택할 수 있게 한다.
- 30초짜리 MP3 클립을 생성한다.
- 브라우저에서 완성된 벨소리 MP3를 다운로드할 수 있게 한다.

## 1차 버전에서 제외할 것

- Android 또는 iPhone 벨소리 자동 지정.
- 사용자 계정 또는 접근 제어.
- 클라우드 호스팅.
- 공개 배포.
- 모바일 앱 백그라운드 동작.
- 고급 파형 편집.
- 플레이리스트 또는 일괄 변환.
- 영구적인 음악 라이브러리 관리.

## 법적 사용 경계

이 앱은 사용자가 다운로드하고 변환할 권리가 있는 콘텐츠를 개인적으로 다루기 위한 도구다. 공개 유튜브 변환 서비스처럼 보이거나 동작하도록 만들지 않는다.

서버에는 인증이 없으므로, 포트가 인터넷에서 접근 가능하다면 외부에 노출된 것으로 간주해야 한다.

## 권장 아키텍처

작은 로컬 웹 애플리케이션으로 만든다.

- 프론트엔드: 로컬 백엔드가 제공하는 브라우저 UI.
- 백엔드: Windows에서 실행되는 Node.js 서버.
- 다운로드 도구: `yt-dlp`.
- 오디오 처리: `ffmpeg`.
- 저장소: 임시 다운로드 파일과 생성된 벨소리 파일을 저장하는 로컬 프로젝트 폴더.

Node.js를 쓰면 UI와 서버를 하나의 프로젝트 안에 두기 쉽고, 백엔드에서 `yt-dlp`와 `ffmpeg`를 자식 프로세스로 실행하기도 편하다.

## 주요 사용자 흐름

1. 사용자가 브라우저에서 앱을 연다.
2. 검색어 또는 유튜브 URL을 입력한다.
3. 입력이 검색어라면 앱이 최대 5개의 후보 영상을 반환한다.
4. 사용자가 영상을 하나 선택한다.
5. 서버가 가장 적절한 오디오 스트림을 다운로드한다.
6. 필요하면 서버가 원본 오디오를 앱 내부 미리듣기용 MP3로 변환한다.
7. 브라우저가 미리듣기 오디오를 불러온다.
8. 사용자가 음악을 재생하며 슬라이더와 숫자 입력으로 시작 시간을 고른다.
9. 사용자가 `30초 MP3 만들기` 버튼을 누른다.
10. 서버가 `ffmpeg`로 30초짜리 MP3 클립을 만든다.
11. 브라우저가 다운로드 버튼을 보여준다.

## UI 설계

1차 버전은 실용적이고 모바일에서 쓰기 쉬운 화면을 목표로 한다.

- 단일 페이지 레이아웃.
- 상단 검색어 또는 URL 입력창.
- 제목, 길이, 채널, 가능한 경우 썸네일이 포함된 결과 목록.
- 현재 선택한 트랙 영역.
- 미리듣기용 기본 오디오 플레이어.
- 시작 시간 슬라이더.
- 초 단위 시작 시간 숫자 입력.
- 생성 버튼.
- 생성 후 다운로드 버튼.
- 다운로드, 변환, 자르기, 오류 상황을 알려주는 간단한 상태 메시지.

파형 편집은 의도적으로 뒤로 미룬다. 1차로 실제 사용할 수 있는 버전에는 기본 오디오 재생과 시작 시간 슬라이더만으로 충분하다.

## 백엔드 엔드포인트

제안하는 엔드포인트는 다음과 같다.

- `GET /`: 웹 UI를 제공한다.
- `POST /api/search`: `{ "query": string }`을 받아 후보 영상을 반환한다.
- `POST /api/prepare`: `{ "url": string }`을 받아 오디오를 다운로드하고 로컬 미리듣기 트랙 id를 반환한다.
- `GET /api/audio/:id`: 준비된 MP3 미리듣기 파일을 스트리밍한다.
- `POST /api/clip`: `{ "id": string, "startSeconds": number }`를 받아 30초짜리 MP3를 만든다.
- `GET /api/download/:clipId`: 생성된 벨소리 파일을 다운로드한다.

모든 id는 앱이 생성한다. 클라이언트가 원시 파일 경로를 직접 넘기게 해서는 안 된다.

## 데이터와 파일 구조

앱 소유의 로컬 폴더를 사용한다.

- `data/source/`: 다운로드한 원본 오디오.
- `data/preview/`: 변환된 미리듣기 MP3 파일.
- `data/clips/`: 생성된 30초 MP3 파일.
- `data/jobs.json`: 나중에 히스토리가 필요해지면 사용할 수 있는 간단한 매니페스트. 더 커지면 SQLite도 고려할 수 있다.

1차 버전에서는 메타데이터를 메모리에 두고 파일만 디스크에 저장해도 충분하다. 서버가 재시작되면 기존 생성 파일 자체는 남아 있을 수 있지만, 간단한 매니페스트를 추가하지 않는 한 예전 다운로드 링크를 보장하지 않는다.

## 로그인 없는 구조에서의 보안 제약

인증을 넣지 않으므로 다음 제약을 둔다.

- 호스트와 포트를 설정 가능하게 한다.
- `localhost`가 아닌 주소에 바인딩할 때는 서버 시작 로그에 경고를 눈에 띄게 출력한다.
- 임의 파일 읽기를 절대 노출하지 않는다.
- 브라우저에서 쉘 조각을 받지 않는다.
- 유튜브 URL과 검색어 길이를 검증한다.
- 검증된 인자로 조립한 고정된 `yt-dlp`, `ffmpeg` 명령만 실행한다.
- 모든 생성 파일은 앱 소유 폴더 안에 저장한다.
- 동시에 실행되는 다운로드와 클립 생성 수를 제한해 실수로 서버가 과부하되지 않게 한다.
- 오래된 임시 파일을 정리하는 기본 청소 기능을 둔다.

이 제약이 앱을 비공개로 만들어주지는 않는다. 다만 모르는 방문자가 서버에 시킬 수 있는 일을 제한한다.

## 오류 처리

앱은 다음 상황에서 이해하기 쉬운 오류를 보여줘야 한다.

- `yt-dlp`가 설치되어 있지 않거나 찾을 수 없음.
- `ffmpeg`가 설치되어 있지 않거나 찾을 수 없음.
- 검색 결과 없음.
- 다운로드 실패.
- 지원되지 않거나 사용할 수 없는 영상.
- 클립 시작 시간이 오디오 길이를 넘어감.
- 서버가 바쁨.

백엔드는 짧은 사용자용 메시지를 담은 구조화된 JSON 오류를 반환한다.

## 테스트 계획

1차 버전은 다음 방식으로 테스트한다.

- URL 판별, 입력 검증, 파일명 정리, 시간 검증 단위 테스트.
- 잘못된 입력과 예상 JSON 형태를 확인하는 백엔드 라우트 테스트.
- `yt-dlp`와 `ffmpeg`가 사용 가능한지 확인하는 스모크 테스트.
- Windows PC 브라우저에서 수동 테스트.
- PC IP 주소를 통해 휴대폰 브라우저에서 수동 테스트.

전체 유튜브 다운로드 자동 테스트는 처음에는 피한다. 느리고 네트워크 상태에 영향을 받기 때문이다.

## 구현 단계

1. Node.js 웹 서버와 정적 UI를 만든다.
2. `yt-dlp`와 `ffmpeg` 환경 확인을 추가한다.
3. URL 입력과 오디오 준비 기능을 구현한다.
4. 키워드 검색을 추가한다.
5. 브라우저 미리듣기 재생을 추가한다.
6. 시작 시간 선택과 30초 자르기를 추가한다.
7. 다운로드 엔드포인트를 추가한다.
8. 정리 작업, 제한, 집중 테스트를 추가한다.
9. Windows PC IP 주소로 휴대폰 브라우저에서 테스트한다.

## 1차 버전 결정사항

- 프론트엔드는 plain HTML, CSS, 브라우저 JavaScript를 사용한다.
- 1차 버전의 서버 메타데이터는 메모리에 유지한다.
- 생성된 파일은 디스크에 남기지만, 서버 재시작 후 예전 다운로드 링크는 보장하지 않는다.
- 파형 편집은 첫 사용 가능 버전 이후로 미룬다.
- 기본 포트는 `3000`으로 하고 설정 가능하게 한다.
- 네트워크의 휴대폰에서 접속하려면 `0.0.0.0`에 바인딩한다.
